# Handoff Checklist — Planning → Implementation / Implementation → Release

Purpose: ensure no lost context during phase handoffs.

- [ ] Owner assigned for the work item (name & role).
- [ ] Clear acceptance criteria and DoD documented in the issue.
- [ ] Dependencies identified and owners assigned.
- [ ] QA strategy defined (unit, integration, manual steps).
- [ ] Monitoring and telemetry requirements specified.
- [ ] Release constraints or windows identified (if applicable).
- [ ] Runbooks and rollback plans drafted (if applicable).
